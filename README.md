Installing python_abi
=====================

Installing `python_abi` from the `e8035669acarmv7` channel can be achieved by adding `e8035669acarmv7` to your channels with:

```
conda config --add channels e8035669acarmv7
conda config --set channel_priority strict
```

Once the `e8035669acarmv7` channel has been enabled, `python_abi` can be installed with `conda`:

```
conda install python_abi
```

or with `mamba`:

```
mamba install python_abi
```

It is possible to list all of the versions of `python_abi` available on your platform with `conda`:

```
conda search python_abi --channel e8035669acarmv7
```

or with `mamba`:

```
mamba search python_abi --channel e8035669acarmv7
```

Alternatively, `mamba repoquery` may provide more information:

```
# Search all versions available on your platform:
mamba repoquery search python_abi --channel e8035669acarmv7

# List packages depending on `python_abi`:
mamba repoquery whoneeds python_abi --channel e8035669acarmv7

# List dependencies of `python_abi`:
mamba repoquery depends python_abi --channel e8035669acarmv7
```




Updating python_abi-feedstock
=============================

If you would like to improve the python_abi recipe or build a new
package version, please fork this repository and submit a PR. Upon submission,
your changes will be run on the appropriate platforms to give the reviewer an
opportunity to confirm that the changes result in a successful build. Once
merged, the recipe will be re-built and uploaded automatically to the
`e8035669acarmv7` channel, whereupon the built conda packages will be available for
everybody to install and use from the `e8035669acarmv7` channel.
Note that all branches in the conda-forge/python_abi-feedstock are
immediately built and any created packages are uploaded, so PRs should be based
on branches in forks and branches in the main repository should only be used to
build distinct package versions.

In order to produce a uniquely identifiable distribution:
 * If the version of a package **is not** being increased, please add or increase
   the [``build/number``](https://docs.conda.io/projects/conda-build/en/latest/resources/define-metadata.html#build-number-and-string).
 * If the version of a package **is** being increased, please remember to return
   the [``build/number``](https://docs.conda.io/projects/conda-build/en/latest/resources/define-metadata.html#build-number-and-string)
   back to 0.
